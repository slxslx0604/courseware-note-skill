# Courseware Note Skill

一个用于将课程课件 PDF 转换为标准化 Obsidian Markdown 学习笔记的 Codex Skill / Prompt 工作流。

本项目适用于需要长期、批量、稳定处理课程课件的场景。它的目标不是简单摘要，而是将课件内容完整重构为便于学习、复习、检索和长期保存的课程笔记。

## 项目定位

很多课程 PDF 课件本身更适合课堂展示，而不适合直接作为复习资料：

- 信息被拆散在多页 PPT 中；
- 同类知识点重复出现；
- 标题层级不稳定；
- 大量内容以短句、图示、表格或项目清单形式存在；
- 直接复制容易形成碎片化笔记；
- 过度总结又容易遗漏知识点。

本项目提供一套标准化处理规则，用于指导 AI 工具将课件 PDF 转换为结构统一、内容完整、阅读体验更好的 Markdown 笔记。

## 核心目标

将课程课件 PDF 转换为适合 Obsidian 使用的 Markdown 学习笔记。


## 适用场景

本项目适合：

- 课程课件 PDF 整理；
- 企业培训课件整理；
- 电力、工程、计算机、管理类课程笔记整理；
- Obsidian 知识库建设；
- 批量 PDF 转 Markdown 学习资料；
- 将课堂展示型材料转化为复习型笔记；
- 构建个人长期学习资料库。

不适合：

- 只想生成极短摘要；
- 需要完全逐字转录课件；
- 希望生成思维导图或 Mermaid 图；
- 希望 AI 大量扩写课件外知识；
- 需要保留原 PDF 视觉版式的场景。

## 项目特点

### 1. 不是摘要，而是结构化重写

本工作流不会将课件压缩成几段总结，而是尽可能保留全部实质知识点，并将其重构为更适合学习的笔记结构。

### 2. 不机械复制 PPT

PPT 中常见的短句、重复标题、分散页面和演示性文字会被重新组织，避免形成松散、割裂的 Markdown。

### 3. 保持内容完整

定义、分类、流程、参数、设备、案例、工程项目、操作要求、安全要求等实质内容都应保留。

### 4. 平衡段落、列表和表格

本项目特别强调：  
不要因为避免碎片化就完全不用分点，也不要为了整齐把所有内容都做成列表。

### 5. 适配 Obsidian

输出 Markdown 会尽量保持标题稳定、链接清晰，并在学习自测中使用 Obsidian 内部链接关联正文知识点。

示例：

```markdown
关联原文：[[课程文件名#对应标题]]
````

## 使用方式一：作为 Codex Skill 使用

将 `SKILL.md` 放入你的 Codex Skills 目录中。

示例目录：

```text
courseware-note/
└── SKILL.md
```

之后在 Codex 中调用：

```text
请使用 courseware-note skill，处理指定目录中的所有 PDF，并输出同名 Markdown 笔记。
```

也可以指定输入输出路径：

```text
请使用 courseware-note skill，处理 `D:\课程课件\待处理PDF` 中的所有 PDF，输出到 `D:\课程课件\输出笔记`。
```

## 使用方式二：在 ChatGPT 网页端使用

如果不使用 Codex，也可以直接使用 `prompt.md`。

步骤：

1. 在 ChatGPT 网页端上传一个或多个课程 PDF；
2. 复制 `prompt.md` 中的完整提示词；
3. 粘贴并发送；
4. 等待生成 Markdown 课程笔记。

适合临时处理少量课件。

## 使用方式三：作为个人笔记工作流规范

你也可以不依赖固定工具，只将本项目作为“课件转笔记”的格式规范。

适合用于：

* 个人 Obsidian 知识库；
* 团队培训资料整理；
* 课程复习资料标准化；
* AI 笔记处理流程设计。

## 输出格式

每个 PDF 应生成一份独立 Markdown 笔记。


Markdown 文件应便于在 Obsidian 中继续修改、链接、复习和归档。

## 局限性

本项目依赖 AI 对 PDF 内容的提取和理解能力，因此可能存在以下问题：

* 扫描版 PDF 识别不完整；
* 图片、图表、流程图可能无法完全转写；
* 复杂表格可能需要人工校对；
* 部分课件中的隐含结构可能识别不准确；
* 学习自测题的链接需要检查是否指向真实标题。

因此，建议对生成结果进行人工复核，尤其是涉及参数、工程名称、安全要求、流程步骤和图表内容的部分。

## 推荐工作流

1. 准备课程 PDF；
2. 使用本 Skill 或 Prompt 生成 Markdown 笔记；
3. 检查标题结构；
4. 检查图表和参数；
5. 检查学习自测链接；
6. 将 Markdown 导入 Obsidian；
7. 根据个人学习习惯继续补充批注、标签和双链。
